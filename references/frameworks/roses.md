# ROSES — 问题解决

**适用场景：** 系统性解决复杂问题，从角色定义到场景描述到执行步骤的完整链条。适合策略制定、方案设计、复杂分析。

**参数数量：** 5

## 模板

```
## ROSES 框架

**Role（角色）**：[AI应扮演的角色身份和专业领域]
**Objective（目标）**：[要解决的核心问题或达成的具体目标]
**Scenario（场景）**：[具体情境描述，让AI充分理解上下文和约束]
**Expected Solution（预期方案）**：[你心中大致的方向、偏好或硬性要求]
**Steps（步骤）**：[建议的执行步骤框架]
```

## 示例

```
## ROSES 框架

**Role（角色）**：互联网行业战略顾问，专注中国科技公司竞争分析

**Objective（目标）**：制定一份字节跳动进军AI搜索市场对百度影响的竞争分析，给出百度的应对策略建议

**Scenario（场景）**：
- 字节跳动凭借抖音流量和豆包大模型，正在测试AI搜索功能
- 百度搜索市场份额从70%下滑至约60%，但文心一言用户增长迅速
- 分析时间窗口：2025-2027年
- 受众：投资分析师，需要数据驱动的结论

**Expected Solution（预期方案）**：
- 重点关注流量迁移路径和用户习惯变化
- 不要仅罗列事实，需给出有洞察的判断
- 使用SWOT框架作为分析基础

**Steps（步骤）**：
    1. 梳理双方核心优势和竞争要素
    2. 分析AI搜索对传统搜索的替代逻辑
    3. 预测三种可能的市场格局演变
    4. 给出百度的最优应对策略
```
