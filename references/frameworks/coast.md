# COAST — 场景驱动

**适用场景：** 需要详细场景描述的任务，上下文信息对输出质量影响巨大。适合用户研究、场景分析、用例设计。

**参数数量：** 5

## 模板

```
## COAST 框架

**Context（上下文）**：[完整的背景信息，包括项目背景、行业环境、相关历史]
**Objective（目标）**：[要达成的最终目标]
**Actions（行动）**：[需要执行的具体操作列表]
**Scenario（场景）**：[使用场景描述、目标受众画像、使用情境]
**Task（任务）**：[拆解后的具体子任务，每个可独立执行]
```

## 示例

```
## COAST 框架

**Context（上下文）**：
我们是一家SaaS公司，产品是面向中小企业的项目管理工具。
目前月活10万，付费转化率仅3%，低于行业平均的5%。
用户调研显示很多用户"不知道怎么用到日常工作中"。

**Objective（目标）**：设计一套新用户Onboarding流程，将7日留存率从20%提升到35%

**Actions（行动）**：
- 分析现有Onboarding流失节点
- 对标Notion/飞书的Onboarding设计
- 设计3套备选方案并评估
- 给出A/B测试建议

**Scenario（场景）**：
- 目标用户：5-20人团队的项目经理/创始人
- 使用情境：第一次注册后，希望能快速上手创建第一个项目
- 痛点：功能多但不知道从哪开始

**Task（任务）**：
    1. 画出当前Onboarding的用户旅程图
    2. 标注每个节点的流失率和原因分析
    3. 设计新的Onboarding流程（含界面文案建议）
    4. 给出核心指标监测方案
```
