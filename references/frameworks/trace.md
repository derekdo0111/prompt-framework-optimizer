# TRACE — 示例驱动

**适用场景：** 需要精确控制输出格式和风格的任务。提供"输入-输出示例"让AI精准理解你的期望。

**参数数量：** 5

## 模板

```
## TRACE 框架

**Task（任务）**：[要完成的具体任务]
**Request（需求）**：[细节要求、风格偏好、约束条件]
**Action（行动）**：[AI应采取的处理方式]
**Context（上下文）**：[背景信息]
**Example（示例）**：[提供1-2组输入-输出示例，让AI精准理解期望]
    ---
    【输入】：
    XXXXXXXXXXXXXXXX
    
    【期望输出】：
    XXXXXXXXXXXXXXXX
    ---
```

## 示例

```
## TRACE 框架

**Task（任务）**：将以下用户反馈按"功能需求/Bug/体验建议"分类，并提取关键信息

**Request（需求）**：
- 每条反馈只归入一个类别
- 提取不超过15字的关键信息摘要
- Bug类还需标注严重程度（高/中/低）

**Action（行动）**：逐条分析反馈内容，根据关键词和语义进行分类和信息提取

**Context（上下文）**：
我们正在做季度用户反馈清理，共有237条需要处理。先给你5条试分类，确认标准一致后批量处理。

**Example（示例）**：
    ---
    【输入】：
    "导出报表的时候经常超时报错，我等了5分钟还没出来"
    
    【期望输出】：
    类别：Bug
    关键信息：报表导出超时错误
    严重程度：高
    ---
    【输入】：
    "希望增加批量删除项目的功能，一个一个删太慢了"
    
    【期望输出】：
    类别：功能需求
    关键信息：增加批量删除项目功能
    严重程度：—
    ---
```
