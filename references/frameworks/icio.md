# ICIO — 结构化输入输出

**适用场景：** 数据处理、格式转换、批量处理等结构化任务。输入和输出都有明确的格式要求。

**参数数量：** 4

## 模板

```
## ICIO 框架

**Instruction（指令）**：[清晰的处理指令，一句话说清楚做什么]
**Context（上下文）**：[为什么需要这个处理，上下游关系]
**Input Data（输入数据）**：[原始数据/文本，保持结构化]
**Output Indicator（输出指标）**：
    ├─ 格式：[JSON / Markdown表格 / YAML / 列表 / ...]
    ├─ 字段：[需要包含哪些字段及其定义]
    ├─ 规则：[转换规则/映射关系/边界条件]
    └─ 样例：[一个期望输出的例子]
```

## 示例

```
## ICIO 框架

**Instruction（指令）**：将以下自由文本格式的财务数据转换为JSON格式

**Context（上下文）**：
这些数据来自各公司的年报摘要，需要统一格式后导入数据库进行横向对比。

**Input Data（输入数据）**：
```
腾讯控股2025年营收6600亿，净利润1800亿，毛利率48%
阿里巴巴2025年营收9000亿，净利润800亿，毛利率38%
字节跳动2025年营收11000亿，净利润1500亿，毛利率42%
```

**Output Indicator（输出指标）**：
    ├─ 格式：JSON数组
    ├─ 字段：company(公司名), year(年份), revenue_bi(营收/亿), net_profit_bi(净利润/亿), gross_margin(毛利率/百分比数值)
    ├─ 规则：
    │   - 所有数值字段提取为数字类型（去除"亿"等单位）
    │   - 毛利率提取为数字（如48%，提取48）
    │   - 如果某字段缺失，设为null
    └─ 样例：
    ```json
    [{"company": "腾讯控股", "year": 2025, "revenue_bi": 6600, "net_profit_bi": 1800, "gross_margin": 48}]
    ```
```
