# C.R.I.S.P.E — 深度对话

**适用场景：** 追求高质量、有深度、有见地的回答。适合研究探讨、创意写作、哲学思辨、战略分析等需要AI"真正思考"的任务。

**参数数量：** 5

## 模板

```
## C.R.I.S.P.E 框架

**Capacity & Role（能力与角色）**：[赋予AI的专业能力和角色定位]
**Insight（洞察）**：[你已有的观点、初步想法、观察，让AI在此基础上升华]
**Statement（声明）**：[明确表达你的立场或核心假设，界定讨论边界]
**Personality（个性）**：[期望的语气和风格：严谨学术/犀利批判/温暖共情/幽默轻松]
**Experiment（实验）**：[邀请AI尝试不同角度或方法，探索多种可能性]
```

## 示例

```
## C.R.I.S.P.E 框架

**Capacity & Role（能力与角色）**：
你是一位既懂技术底层又懂商业逻辑的AI行业观察者，有10年以上的行业研究经验。

**Insight（洞察）**：
我注意到一个现象：大模型能力越强，SaaS公司的护城河反而越弱。
过去SaaS靠"工作流+数据沉淀"建立壁垒，但AI时代用户可能直接说"帮我做XX"，
不再点击菜单。这意味着SaaS正在从"工具"变成"能力调用层"。

**Statement（声明）**：
我的核心假设是：未来5年，传统SaaS的商业模式会被AI原生应用颠覆，
但垂直领域的SaaS（如医疗、法律）因为合规和数据独占性，反而会更强。

**Personality（个性）**：
请用犀利的风格，敢于挑战主流观点。如果有不同意见，直接反驳我。
不要"一方面...另一方面"的圆滑表述。

**Experiment（实验）**：
尝试从三个不同视角分析：
1. 技术视角：Agent能力演进如何改变软件交互范式
2. 商业视角：订阅制定价模型会不会被"按结果付费"取代
3. 反方视角：为什么SaaS不会死？（请挑战我自己的假设）
```
