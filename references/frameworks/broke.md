# BROKE — 迭代优化

**适用场景：** 需要多轮打磨的复杂任务，如产品设计、策略文档、创意方案。强调"先出初稿，再反馈迭代"的工作方式。

**参数数量：** 5

## 模板

```
## BROKE 框架

**Background（背景）**：[完整的项目背景、起因、现状描述]
**Role（角色）**：[AI的角色定位]
**Objective（目标）**：[核心目标和成功标准，越具体越好]
**Key Results（关键结果）**：[可衡量的具体产出指标]
**Evolve（迭代改进）**：
    第一轮：基于以上信息生成初稿
    我的反馈：[留空，待用户填写]
    第二轮：根据反馈优化
    ...持续迭代直到满意
```

## 示例

```
## BROKE 框架

**Background（背景）**：
我们是一家B2B SaaS公司，准备发布AI功能模块。
目标客户是中型企业的HR部门，竞品Workday和北森已推出类似功能。
我们需要一份产品发布策略，兼顾市场声量和实际转化。

**Role（角色）**：你是一位SaaS产品市场专家，熟悉B2B产品发布全流程

**Objective（目标）**：制定AI功能模块的产品发布策略，确保上线首月获取500个试用客户

**Key Results（关键结果）**：
- KR1：发布前2周完成3篇预热内容（行业报告+客户案例+产品演示视频脚本）
- KR2：联合至少2家行业媒体做首发报道
- KR3：上线首周安排5场面向HR社群的直播Demo

**Evolve（迭代改进）**：
    第一轮：请根据以上信息，先输出发布策略的框架和核心创意方向
    用户的反馈：[待填写]
    第二轮：根据反馈深化具体执行方案
```
