# CARE — 结果导向

**适用场景：** 以结果反推的思维方式，适合需要明确交付物的任务。先描述期望的最终产出，再反推需要做什么。

**参数数量：** 4

## 模板

```
## CARE 框架

**Context（上下文）**：[背景和动机，为什么需要这个结果]
**Action（行动）**：[需要执行的任务]
**Result（结果）**：[具体要产出的交付物描述，越详细越好]
**Example（示例）**：[参考范例，展示理想输出长什么样]
```

## 示例

```
## CARE 框架

**Context（上下文）**：
下周要向CEO汇报Q2产品规划，需要一份有说服力的一页纸摘要。
CEO风格是：只看数据和结论，不读长篇大论。

**Action（行动）**：将30页的产品规划文档压缩为一页纸的执行摘要

**Result（结果）**：
- 上半部分：3个核心指标（当前值 + Q2目标值）
- 中间：3个重点项目的时间线和资源需求
- 下半部分：1个主要风险 + 缓解方案
- 整页不超过500字

**Example（示例）**：
类似亚马逊的"六页纸"风格，但压缩到一页：
"[项目名] — 当前[X]%完成 → Q2目标[Y]% → 需要[Z]资源"
```
