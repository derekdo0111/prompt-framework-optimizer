# 多场景案例库

每个案例展示：原始用户输入 → 信号提取 → 追问 → 匹配框架 → 优化后的Prompt

---

## 案例1：数据分析 → RISE

**用户输入：**
> 帮我把这几个季度的营收数据做个分析，看看趋势怎么样

**Phase 1 信号提取：**
- 复杂度：MEDIUM（"分析"+"趋势"）
- 结构化：MEDIUM（有数据输入）
- 角色：NONE
- 示例：NONE

**Phase 2 追问：**
- Q1 → 用户选B（有点复杂）
- Q2 → 用户选C（没格式要求）
- Q3 → 用户选A（财务分析师）

**匹配框架：RISE**

**优化后的Prompt：**

```
你是一位资深财务分析师，擅长从数据中发现趋势和异常。

【输入】以下各季度营收数据需要分析：
[用户提供的数据]

【分析步骤】
1. 先计算各季度同比/环比增长率
2. 识别增长趋势（加速/减速/稳定）
3. 标注异常波动点并推测可能原因
4. 给出未来1-2个季度的趋势预判

【输出期望】
- 以表格呈现计算数据
- 趋势分析用折线图描述（文字说明走势）
- 异常点必须附带具体数值和可能原因
```

## 案例2：快速问答 → TAC

**用户输入：**
> 什么是ROE？

**Phase 1 信号提取：**
- 复杂度：LIGHT（简单定义类问题）
- 结构化：NONE
- 角色：NONE

**无需追问，直接匹配：TAC**

**优化后的Prompt：**

```
**Task（任务）**：解释ROE（净资产收益率）是什么
**Action（动作）**：用通俗的语言，从定义、公式、实际含义三层递进解释
**Goal（目标）**：让没有财务背景的人也能理解ROE的核心含义
```

## 案例3：格式转换 → ICIO

**用户输入：**
> 把这些股票代码和名称整理成JSON格式，key用symbol和name

**Phase 1 信号提取：**
- 复杂度：LIGHT-MEDIUM
- 结构化：HIGH（"JSON格式"+"key用symbol和name"）
- 角色：NONE

**无需追问，直接匹配：ICIO**

**优化后的Prompt：**

```
**Instruction（指令）**：将以下股票代码和名称列表转换为JSON格式
**Context（上下文）**：用于导入量化分析系统，需要标准化的数据结构
**Input Data（输入数据）**：
[用户提供的股票列表]
**Output Indicator（输出指标）**：
    ├─ 格式：JSON数组
    ├─ 字段：symbol(股票代码), name(公司名称)
    └─ 样例：[{"symbol": "000001", "name": "平安银行"}]
```

## 案例4：策略方案 → ROSES

**用户输入：**
> 帮我制定一个个人投资组合的配置策略，我有50万，风险偏好中等

**Phase 1 信号提取：**
- 复杂度：HEAVY（"策略"+"配置"）
- 结构化：NONE（未指定格式）
- 角色：NONE（未指定专业角色）
- 场景：有（"个人投资"+ "50万" + "风险偏好中等"）

**Phase 2 追问：**
- Q3 → 用户选A（理财顾问）

**匹配框架：ROSES**

**优化后的Prompt：**

```
**Role（角色）**：资深理财顾问，擅长个人资产配置

**Objective（目标）**：为一位中等风险偏好的投资者制定50万资金的配置策略

**Scenario（场景）**：
- 资金量：50万人民币
- 风险偏好：中等（能接受10-15%的最大回撤）
- 投资期限：[请补充：短期/中期/长期]
- 流动性需求：[请补充：是否需要随时可取用]

**Expected Solution（预期方案）**：
- 避免过度集中，单资产不超过20%
- 包含至少3个不同类别的资产
- 考虑当前市场环境（2025年A股震荡、债市收益率下行）

**Steps（步骤）**：
    1. 确定资产大类及配置比例（股票/债券/现金/另类）
    2. 在每个大类下推荐具体标的类型（如指数基金、国债ETF等）
    3. 说明配置逻辑和风险控制机制
    4. 给出再平衡频率建议
```

## 案例5：迭代创作 → BROKE

**用户输入：**
> 帮我写一篇公众号文章，讲AI对普通人的影响，先出个大纲，我看了再改

**Phase 1 信号提取：**
- 复杂度：MEDIUM
- 迭代信号：HIGH（"先出个大纲，我看了再改"）
- 角色：NONE

**无需追问，直接匹配：BROKE**

**优化后的Prompt：**

```
**Background（背景）**：
公众号定位：科技类，面向普通读者（非技术背景），风格通俗易懂。
主题：AI对普通人生活和工作影响的深度分析。

**Role（角色）**：你是科技类公众号主编，擅长用故事和案例把复杂技术讲得人人都懂

**Objective（目标）**：产出一篇2000字左右的公众号文章，阅读量目标5000+

**Key Results（关键结果）**：
- KR1：开篇300字内吸引读者继续阅读（有具体故事或数据钩子）
- KR2：包含至少3个普通人能理解的AI应用案例
- KR3：结尾有明确的观点和行动建议

**Evolve（迭代改进）**：
    第一轮：请先输出文章大纲（含每部分的核心观点和案例方向）
    用户的反馈：[待第一轮输出后填写]
    第二轮：根据反馈写出完整初稿
```

## 案例6：深度思辨 → C.R.I.S.P.E

**用户输入：**
> 我想和你深度讨论一个问题：量化交易会不会让市场变得更有效还是更无效？

**Phase 1 信号提取：**
- 复杂度：HEAVY（"深度讨论"+"XXX还是XXX"思辨型问题）
- 结构化：NONE
- 角色：NONE（但话题本身需要专家视角）

**Phase 2 追问：**
- Q3 → 用户选A（量化金融专家）

**匹配框架：C.R.I.S.P.E**

**优化后的Prompt：**

```
**Capacity & Role（能力与角色）**：
你是一位量化金融领域的资深研究员，同时具备学术理论功底和实战经验。

**Insight（洞察）**：
我观察到一个矛盾：量化基金规模越大，超额收益越难做（说明市场在变有效），
但2020年疫情和2024年AI行情中，量化基金的表现分化巨大（说明不是简单的"有效"）。
似乎存在一种"动态均衡"。

**Statement（声明）**：
我的立场是：量化交易在微观层面（日内/周度）提升了市场有效性，
但在宏观层面（年度/跨资产）可能制造了新的无效（比如因子拥挤导致的崩盘）。

**Personality（个性）**：
请用学术探讨的语气，但不要堆砌公式。逻辑严密但不枯燥。
可以对我的观点提出质疑。

**Experiment（实验）**：
请从三个视角展开：
1. 有效市场假说视角：套利行为如何消除定价错误
2. 行为金融学视角：量化交易的"羊群效应"如何创造新套利机会
3. 你的独立判断：未来5年，哪种力量会占主导？
```
